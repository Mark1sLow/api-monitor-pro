# Техническое задание на проект: API Monitor Pro

## 1. Цель проекта

Создание веб-сервиса для мониторинга и анализа ключевых метрик производительности API-методов (SLI/SLO): доступность, задержка ответа, частота ошибок. Система позволяет DevOps-инженерам отслеживать ухудшения, формировать отчеты и получать уведомления по email при нарушении установленных порогов.

## 2. Роли пользователей

- **Гость**: Просмотр общих метрик (например, статус-страница), ознакомление с отчетами.
- **Зарегистрированный пользователь**: Создание и управление своими проектами и методами API, настройка SLI/SLO, просмотр детализированных графиков, получение уведомлений.
- **Администратор**: Управление пользователями, доступ к системным отчетам и логам.

## 3. Модели данных (сущности)

- **Project (Проект)**: Группировка API-методов.
  - Поля: `name`, `url`, `owner` (ForeignKey на User), `created_at`.
- **Endpoint (Эндпоинт)**: Один метод API, который мониторится.
  - Поля: `project` (ForeignKey), `method` (GET, POST и т.д.), `path`, `sla_latency_ms`, `sla_error_rate`.
- **Measurement (Измерение)**: Запись результата проверки.
  - Поля: `endpoint` (ForeignKey), `timestamp`, `response_time_ms`, `status_code`, `is_error`, `sla_breached`.

## 4. Ключевой функционал (User Stories)

- **Добавление проекта и эндпоинта**: Пользователь указывает URL проекта и методы API, которые нужно мониторить, задает SLA-параметры.
- **Просмотр графиков SLI**: На странице эндпоинта отображаются графики задержки, ошибок и доступности за последние 7 дней.
- **Уведомления о нарушениях**: При превышении лимитов (например, ошибка > 5% или задержка > 500ms) отправляется уведомление на email.
- **Генерация отчета**: Пользователь может сформировать отчет по SLI/SLO за выбранный период (PDF/HTML).
- **Автоматическая проверка**: Система по расписанию (Cron или Celery Beat) выполняет запросы к API и сохраняет результаты.

## 5. Внешние интеграции / Аналитика

- **Интеграции**: SMTP (email-уведомления), HTTP-запросы к API-методам.
- **Аналитика**: Использование **Pandas** для расчета средней задержки, количества ошибок, соблюдения SLO за период. Визуализация графиков через **Plotly** или **Matplotlib**.
